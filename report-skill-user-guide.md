# 云助手报表查询技能使用说明

这份说明是给普通使用者看的。目标只有两件事：

1. 把技能装到 OpenClaw 里
2. 知道平时怎么问，能查到自己想看的经营数据

---

## 一、这个技能能做什么

安装后，你可以直接在 OpenClaw 里用自然语言提问，例如：

- 查询今天营业额
- 查询本周商品分类销售排行
- 这个月新增了多少会员
- 按业务类型看一下今天的数据
- 查询支付分组占比

常见可查询内容包括：

- 营业额、实收、优惠、订单数、客流、退款
- 商品销售排行、商品分类销售排行
- 会员总数、新增会员数
- 支付分组 / 支付科目
- 业务类型分析，例如堂食、外卖、退款

---

## 二、安装前先确认

开始前，请先确认这台电脑已经满足下面两点：

1. 已安装并能正常打开 OpenClaw
2. OpenClaw 的本地服务可正常使用

如果你之前已经在这台机器上正常用过 OpenClaw，一般就可以直接继续。

---

## 三、怎么安装

下面给两种方式。选一种就行。

### 方式 A：适合会用终端的人

这是最直接、最稳定的安装方式。

#### 第 1 步：下载技能仓库

在终端里执行：

```bash
git clone https://github.com/liangxu1186-1/openclaw-skills.git
```

如果你之前已经下载过这个仓库，就先进入原目录再更新：

```bash
cd openclaw-skills
git pull
```

#### 第 2 步：创建 OpenClaw 的 skills 目录

```bash
mkdir -p ~/.openclaw/skills
```

#### 第 3 步：把技能复制到 OpenClaw

```bash
cp -R openclaw-skills/generate_binding_qr ~/.openclaw/skills/
cp -R openclaw-skills/report-skill ~/.openclaw/skills/
```

这两条命令的含义是：

- `generate_binding_qr`：用于生成绑定二维码
- `report-skill`：用于查询报表和经营数据

#### 第 4 步：重启 OpenClaw 服务

```bash
openclaw gateway restart
```

做到这里，安装就完成了。

---

### 方式 B：适合不会用 Git 的人

如果你不会用 Git，也不想自己敲命令，最简单的方式是：直接把一段说明发给 OpenClaw，让它代你执行安装。

#### 你可以直接发这段话给 OpenClaw

```text
请帮我下载安装全来店报表数据技能，并重启服务
https://github.com/liangxu1186-1/openclaw-skills.git
mkdir -p ~/.openclaw/skills
cp -R openclaw-skills/generate_binding_qr ~/.openclaw/skills/
cp -R openclaw-skills/report-skill ~/.openclaw/skills/
openclaw gateway restart
```

#### 这段话的意思

OpenClaw 会按这个顺序做事：

1. 下载技能仓库
2. 创建 `~/.openclaw/skills` 目录
3. 复制 `generate_binding_qr` 和 `report-skill`
4. 重启本地服务

#### 使用这种方式前要知道

1. 这台电脑上需要已经装好 OpenClaw
2. OpenClaw 需要有权限执行本地命令
3. 最好把整段文字原样发给它，不要删掉命令

#### 安装完成后怎么确认

你可以继续对 OpenClaw 说：

```text
请帮我确认全来店报表数据技能是否已经安装成功
```

或者直接测试：

```text
查询今天营业额
```

## 四、怎么确认安装成功

安装完成后，可以这样确认。

### 方法 1：直接问一句

在 OpenClaw 里输入：

```text
查询今天营业额
```

或者：

```text
生成绑定二维码
```

如果能正常返回结果，说明技能已经生效。

### 方法 2：查看技能目录

终端执行：

```bash
ls ~/.openclaw/skills
```

正常情况下，至少能看到：

- `generate_binding_qr`
- `report-skill`

## 五、日常怎么用

### 1. 查询营业数据

你可以这样问：

```text
查询今天营业额
今天卖了多少
本周营业情况怎么样
上月营业额是多少
```

一般会返回：

- 实收金额
- 订单金额
- 优惠金额
- 订单数
- 客流量
- 退款金额
- 简短分析

### 2. 查询商品销售排行

你可以这样问：

```text
查询今天商品分类销售排行
本周哪些商品卖得最好
上周商品销售情况
这个月热销商品有哪些
```

一般会返回：

- 分类销售排行
- 热销商品
- 销售额 / 销量 / 实收
- 简短经营分析

### 3. 查询会员数据

你可以这样问：

```text
目前有多少会员
这个月新增了多少会员
上月新增会员数
```

### 4. 查询支付和业务类型

你可以这样问：

```text
查询支付分组
按业务类型查一下
今天堂食和外卖分别是多少
```

## 六、时间范围怎么说

下面这些说法都可以直接用：

| 你说的话 | 系统通常理解为 |
| --- | --- |
| 今天 | 今天 00:00 到现在 |
| 昨天 | 昨天全天 |
| 本周 | 本周一到现在 |
| 上周 | 上周一到上周日 |
| 本月 | 本月 1 号到现在 |
| 上月 | 上个月完整自然月 |
| 3月23日到3月29日 | 指定日期范围 |

如果你把时间说清楚，返回通常会更稳定、更快。


原因很简单：问题越具体，技能越容易选对接口，第一次就查准。

## 八、常见问题

### 1. 为什么有时查询比较慢？

常见原因：

- 模型能力，思考速度
- 网络延迟
- 查询范围太大
- 返回数据太多

建议：

- 尽量先查一天、一周这种较短范围
- 先问明确问题，不要一句话里塞太多要求

### 2. 为什么结果里有些分类看起来不规范？

例如“临时品”之类的分类，通常是业务系统原始数据里的分类方式决定的，不一定是技能生成的。

### 3. 为什么提示查询异常？

优先检查：

- 时间范围是不是太大
- 当前门店 / 组织是不是正确
- SaaS 报表服务当时是否可用

## 九、更新技能的方法

如果以后技能有新版，普通用户最省事的方式不是自己区分 Git 或 ZIP，而是直接让 OpenClaw 帮你更新。

### 你可以直接发这段话给 OpenClaw

```text
请帮我更新全来店报表数据技能，并在更新完成后重启服务
https://github.com/liangxu1186-1/openclaw-skills.git
mkdir -p ~/.openclaw/skills
cp -R openclaw-skills/generate_binding_qr ~/.openclaw/skills/
cp -R openclaw-skills/report-skill ~/.openclaw/skills/
openclaw gateway restart
```

### 这段话会做什么

1. 重新获取最新的技能文件
2. 覆盖本机已有的 `generate_binding_qr`
3. 覆盖本机已有的 `report-skill`
4. 重启 OpenClaw 本地服务

### 更新后怎么确认

你可以继续对 OpenClaw 说：

```text
请帮我确认全来店报表数据技能是否已经更新成功
```

或者直接测试一个最常用的问题：

```text
查询今天营业额
```
